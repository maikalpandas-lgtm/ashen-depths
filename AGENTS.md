# Agent rules — Ashen Depths

## Art / sprites (обязательно)

1. **Не генерировать** новые картинки (image_gen / Imagine / «перерисуй») **без явной просьбы** пользователя.
2. Работать с **уже существующими** ассетами из каталога:
   - `assets/textures/ASSETS.md` — подписи, назначение, **рецепт обрезки**
   - `assets/textures/masters/` — source + PNG masters
   - `assets/textures/*.png` — файлы, которые грузит игра
3. Если нужно поправить спрайт: **scale / position / cut / код** — не новая генерация.
4. Новые ассеты — **только** после фразы вроде «нарисуй / сгенерируй / перерисуй X».
5. После gen (когда попросили): подписать в `ASSETS.md` + masters, не плодить безымянные `_tmp*`.
6. **Обрезка:** только рабочий способ из `ASSETS.md` § «Как обрезать» — edge flood chromakey (`tools/sprite_cutter.py`), **без** жирного outline-inflate и без агрессивного scissor на стали.
7. **Viewmodel:** не stretch/shake всю руку с факелом — рука статична; лёгкий stretch только у мелких wall-torch props.

## Git (обязательно)

1. **После каждого осмысленного изменения** — сразу `git commit` + **`git push origin main`** на GitHub.
2. Не копить кучу правок «на потом» — одно логическое изменение ≈ один коммит.
3. Сообщения коммитов: коротко **что** и **зачем** (en или ru), без мусора.
4. Не коммитить: `_tmp*`, `raw/cut/`, `gen_*`, keystore/password — см. `.gitignore`.
5. Не force-push на `main` без явной просьбы пользователя.

## Movement / props

- **Grid crawler only:** шаг клетка вперёд/назад, поворот **90°** (A/D), камера **lock forward** (уже `player_controller.gd`).
- **Интерьер (факелы и т.п.) — fixed:** без camera-billboard; ориентация к коридору / стене, при необходимости `flip_h` для зеркала. Не крутить props к камере.
- 🎨 **КОНЦЕПТ: все ОБЪЕКТЫ — 2D-спрайты. 3D только карта** (стены / пол / потолок). Факелы, пропсы, враги, лут — спрайты. **Не** пересобирать проп в меши «чтобы был объём» — объём даётся хало, эмиссией и светом на скале.
- **Факел «смотрит» на игрока** через fixed-Y billboard. Билборд делает **шейдер** (`billboard_y` в `flame_shimmer.gdshader`), т.к. `material_override` глушит `Sprite3D.billboard`.
- **Хало у факела — только радиальный falloff** (процедурный, альфа→0 внутри квада). Непрозрачные/untextured квады на стене дают видимый прямоугольник.

## Свет / атмосфера (референс Spice Mines)

- Тёплое пятно факела **узкое**: скала в паре клеток должна остаться холодной тил. 400 факелов с `energy 5+ / range 9+` заливают пещеру в жёлтый — не поднимать обратно.
- Даль уходит почти в чёрный через `fog_depth_end ~8.5`. Видимость держится ambient'ом, а не радиусом факелов.
- Стык стены и пола — тёмная контактная полоса (`_add_floor_wall_skirting`). Её надо **драпировать по реальной высоте пола** (`_floor_height`): плоская полоса на y≈0 тонет под полом, который у стены поднимается на ~0.16.

## Perf

- Текстуры скалы генерятся процедурно на **каждый** данж. Не звать `_build_materials()` лишний раз и не сканировать все точки решётки на пиксель (`_angular_rock` смотрит только блок 5×5).

## Godot

- **Не запускать игру для проверки** — юзер запускает сам и присылает скрин. Максимум `godot --headless --check-only --script <file>`.

## Geometry / стены

- Панель стены = **ровно `cell_size`** по ширине, без overlap (overlap → z-fighting полосы на стыках).
- Смещение вершин в `_wall_pt` считать **только от мировой позиции**. Panel-local `along` даёт разные значения по обе стороны стыка → щели в пустоту.
- Заваливать рельеф в плоскость (`amp → 0`) **только на углах**, где панель не продолжается (`_wall_face_continues`). На продолжающемся стыке amp = 1 с обеих сторон.
